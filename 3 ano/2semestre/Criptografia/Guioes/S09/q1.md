Se os valores hexadecimais forem iguais, o par é válido.

Num par de chaves RSA, o certificado contém a chave pública, e o ficheiro contém a chave privada. Ambas partilham o mesmo n, onde n = p*q e este p e q são dois numeros primos gigantes, que é a base matemática da relação entre elas.
A chave pública é composta por (n, e),a chave privada é composta por (n, d).

Como o n é comum, se extrairmos este valor de ambos os ficheiros e eles coincidirem, temos a certeza absoluta de que a chave privada em sua posse é a unica chave que funciona com a chave pública que está dentro do certificado.

Utilizando o comando openssl x509 -text -noout -in ALICE.crt obtemos o conteudo do certificado da Alice, agora olhando para o campo Modulus ou seja o nosso n.00:ba:c7:04:67:af:4f:34:df:56:3d:bf:5c:34:94:......

Vamos ter de comparar este n com o n da Alice.key agora se eles foram iguais significa que os pares são validos.

Fazendo openssl rsa -text -noout -in ALICE.key e colocando a passe 1234 que sabemos do enunciado, obtemos o conteudo da key e olhando para o campo Modulus, temos.
00:ba:c7:04:67:af:4f:34:df:56:3d:bf:5c:34:94:.....

Posto isto, verificamos que ambos os n são iguais logo os pares são validos.
